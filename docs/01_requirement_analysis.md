# 需求分析

## 赛题要求拆解

当前赛题定位为“面向操作系统课程的操作系统竞赛和实验”。根据项目理解，需求可初步拆解为：

- 面向教学：实验内容需要适合低年级学生循序渐进学习。
- 面向竞赛：实验体系需要帮助学生理解 OS 功能挑战赛常见任务和工程流程。
- 面向实践：每个实验应包含明确任务、代码入口、测试方式和预期产物。
- 面向复盘：项目应保留过程记录、问题记录、AI 使用记录和测试报告。

## 我们的对应策略

1. 建立清晰的实验主线：从 lab0 环境配置到 lab5 最终集成，覆盖 OS 入门核心路径。
2. 控制实验粒度：每个 lab 保持可独立阅读、可独立验证，避免一次性引入过多概念。
3. 强化测试意识：从 lab1 开始为每个实验准备测试说明和测试用例目录。
4. 记录真实过程：所有测试结果、问题定位和 AI 辅助内容都进入文档记录。
5. 保持合规：外部资料只做引用说明和必要摘录，代码复用前确认许可证。
