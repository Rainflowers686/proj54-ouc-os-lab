# 实验体系总体设计

## 目标

本文说明 OUC xv6 Lab Kit 的实验结构、教学递进和实现边界。设计目标是让学生从可运行的最小实验进入真实内核路径，并能在每一步留下可复现证据。

## 适用对象

本文件适合教师、助教、项目维护者和希望理解课程设计的学生。若只想完成作业，应优先阅读各 `labs/<lab>/README.md`。

## 内容范围

实验体系覆盖 Lab0 到 Lab5。Lab0 验证环境和 baseline；Lab1 进入系统调用；Lab2 观察进程；Lab3 观察页表映射和结构体 copyout；Lab4 观察 fd/file 关系和 fd 元数据；Lab5 整合为 capstone 复现实验。

## 递进关系

Lab1 让学生知道用户态调用如何经 `usys.pl`、`ecall` 和 `syscall.c` 到达 `sys_*`。Lab2 把同一条路径接到 `struct proc` 和锁。Lab3 把观察对象换成用户页表，强调地址空间大小不等于已映射页数。Lab4 把观察对象换成当前进程 fd table 与全局 file table。`memstat` 和 `fdinfo` 用 `argaddr/argint + copyout + struct ABI` 补上真实内核接口常见的数据返回方式。

## 实现组织

独立 patch 服务单 lab 教学，通常从 clean baseline 单独应用。integrated patch sequence 服务综合演示，统一 syscall 编号 22 到 30，避免独立 patch 之间的编号冲突。学生第一次学习应按 lab 走，评审和复现应按 integrated 走。

## 质量标准

每个实验都应有教程、任务书、验证命令、常见卡点和边界说明。测试程序应真实计算现象，不应打印写死结果。验收只匹配稳定文本和 delta，不固定受调度或环境影响的数字。

## 边界条件

本项目是观察型入门实验包，不修改调度策略，不实现完整内存管理，不实现完整文件系统。Lab5 是综合复现和报告训练，不新增内核机制。

## 结构规范

文档应按“背景或问题、过程或设计、证据或命令、风险和后续动作”的顺序组织。历史文档可保留阶段性记录，但必须避免覆盖 current final。

## 语言风格

使用中文技术写作风格，命令、文件名、commit、syscall 名和 SHA256 保持原样。结论应有证据支撑，不使用宣传性、绝对化或无法验证的表述。
