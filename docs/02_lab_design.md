# 实验体系总体设计

## 总体说明

当前实验主线暂定为 xv6-riscv，但本轮不引入 xv6 源码。本文档用于定义 lab0-lab5 的教学目标、输入材料、任务边界、测试方式和评价标准，为后续引入 baseline、编写代码框架和参考实现做准备。

设计原则：

- 面向低年级学生，先解决环境和代码阅读门槛。
- 每个 lab 都要有明确任务、预期产物和测试方式。
- 所有未验证内容标注 TODO 或待验证。
- 参考 rCore/uCore 等课程资源的组织方式，但不复制大段外部资料。
- 后续引入第三方源码时必须保留许可证和来源说明。

## MVP 强保范围与优先级

lab0-lab5 是**完整规划**，但**不代表初赛都要完成**。为在 6/30 前交付可信成果，明确优先级如下：

| 优先级 | 范围 | 初赛要求 | 说明 |
| --- | --- | --- | --- |
| P0 强保 | lab0、lab1 | 必须真实跑通并留证据 | 这两条是 MVP 的可信度底座 |
| P1 二选一 | lab2 或 lab4 | 选其一做设计 + 代码阅读深化，不强求完整实现 | 视 lab1 完成后剩余时间决定 |
| P2 扩展 | lab3、lab5 | 仅保留规划与占位，初赛不强求 | 作为后续迭代方向 |

因此本文档中 lab2-lab5 的「当前状态」标注为规划/占位是**有意为之**，不应被理解为初赛交付清单。每个 lab 新增「验收证据」字段，用于明确「凭什么证明这个 lab 达标」——只有 P0 的两条在初赛阶段必须填出真实证据。

## lab0：环境搭建与工具链检查

| 项目 | 内容 |
| --- | --- |
| 教学目标 | 让学生理解 OS 实验所需的基础工具链，能够检查 Git、bash、make、QEMU 和 RISC-V 编译器是否可用 |
| 前置知识 | Windows/WSL 基础、Linux 命令行、Git 基本操作 |
| 输入材料 | 本仓库、lab0 README、`scripts/check-env.sh` |
| 实验任务 | 安装或确认 WSL2 Ubuntu、Git、make、QEMU、RISC-V toolchain；运行环境检查脚本；记录环境问题 |
| 预期产物 | 环境检查输出、问题记录、后续 baseline 跑通记录 TODO |
| 测试方式 | 运行 `bash scripts/check-env.sh`，记录 `[OK]` 和 `[WARN]` 项 |
| 评价标准 | 能说明本机缺少哪些依赖；不把 WARN 误写成已跑通 xv6 |
| 验收证据（P0 强保） | 真实 `check-env.sh` 输出 + 一台机器上的 xv6 构建/启动日志（文本或截图）+ 环境问题记录；初赛必须填出真实证据 |
| 当前状态 | 文档初版，baseline 跑通步骤待补充 |
